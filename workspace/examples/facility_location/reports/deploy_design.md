# 施設配置最適化 -- 運用設計書

**作成日**: 2026-03-30
**対象**: 関東エリア倉庫配置（10候補地 -> 30小売店、CFL 4施設体制）

---

## 1. 実行概要

| 項目 | 設定 |
|------|------|
| 実行頻度 | **四半期** または **オンデマンド**（新規出店・需要変動時） |
| 計算時間 | CFL ソルバー 約60秒 |
| データソース | CSV手動配置（`data/` フォルダ） |
| 出力 | 施設計画JSON + サマリーレポート |
| 実行スクリプト | `scripts/run_pipeline.py` |

### 実行タイミング

```
定期: 四半期ごとに需要データを更新して再最適化
臨時: 以下のトリガー発生時
```

---

## 2. パイプライン構成

```
Step 1: データ読み込み（candidates.csv, customers.csv, constraints.csv）
  |
Step 2: バリデーション（座標・需要・容量・コストの妥当性チェック）
  |
Step 3: CFL ソルバー（PuLP）で最適化
  |  失敗時 -> UFL フォールバック
Step 4: 結果検証（全顧客割当・容量違反なし・カバレッジ確認）
  |
Step 5: 施設計画JSON + サマリー出力
```

### CFL ソルバーの定式化

- **決定変数**: y[f] = 施設fを開設するか、x[f,c] = 顧客cを施設fに割当
- **目的関数**: 固定費 + 輸送費（Haversine x 道路係数1.3 x 150円/km）の最小化
- **制約**: 各顧客は1施設に割当、容量制約、開設施設からのみサービス

## 3. バリデーションルール

| チェック項目 | 条件 | アクション |
|------------|------|----------|
| 候補施設データ空 | 0件 | **中断** |
| 顧客データ空 | 0件 | **中断** |
| 座標が関東圏外 | 緯度 35.0-37.0, 経度 138.5-141.0 の範囲外 | 警告 |
| 需要 <= 0 | 0以下 | **中断** |
| 容量 <= 0 | 0以下 | **中断** |
| 固定費 <= 0 | 0以下 | **中断** |
| 総需要 > 総容量 | 全施設を開設しても需要を賄えない | **中断** |

## 4. 最適化手法

**CFL ソルバー（PuLP + HiGHS/CBC）**:

- 容量制約付き施設配置問題
- 固定費 + 輸送費の合計を最小化
- 各施設の容量を超えない割当を保証

### 期待される結果（現行データ基準）

| 指標 | 値 |
|------|-----|
| 開設施設数 | 4 |
| 開設施設 | W01, W06, W09, W10 |
| 月額コスト | 約312万円 |
| 容量違反 | 0件 |
| 全顧客割当 | 30/30 |

## 6. 再最適化トリガー

以下のいずれかが発生した場合、パイプラインを再実行する。

- [ ] 新規店舗の出店（顧客データに追加）
- [ ] 既存顧客の需要が20%以上変動
- [ ] 施設の固定費が変更（賃料改定等）
- [ ] 新規候補地の追加
- [ ] 施設の閉鎖・容量変更
- [ ] 四半期の定期見直し

---

## 7. フォールバック手順

### パターン1: CFL が実行不能（Infeasible）

```
原因: 総需要 > 利用可能容量、または制約が厳しすぎる
対応:
  1. パイプラインが自動で UFL フォールバックを実行（容量制約を外す）
  2. UFL も失敗 -> exit code 1 でパイプライン停止
  3. 手動対応: 候補地の追加、または容量の見直し
```

### パターン2: カバレッジ不足

```
原因: 遠距離顧客が存在（水戸ショップ等）
対応: 候補地を追加して再最適化、またはカバレッジ要件の緩和を検討
```

### パターン3: 需要急増

```
原因: 季節変動、キャンペーン等
対応: 5施設体制への拡張を検討（月額+45万円の試算あり）
```

## 8. 前提条件と注意事項

| 項目 | 値 | 備考 |
|------|-----|------|
| 距離計算 | Haversine x 1.3 | 実走行距離ではなく概算 |
| 輸送コスト | 150円/km | 車種・頻度によらず一律 |
| 需要 | 月間固定 | 季節変動は未考慮 |
| ソルバー制限時間 | 60秒 | 大規模化時は延長を検討 |

実データ（Google Maps API距離、実績コスト等）が入手できれば精度が向上する。
