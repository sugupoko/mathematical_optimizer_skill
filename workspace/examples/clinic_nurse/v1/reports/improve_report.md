## 1. 改善戦略

Baseline で HC1-HC8 をすべて満たす feasible 解に到達済みなので、
improve では **SC1-SC6 を目的関数に組み込んで品質を最大化**する。

ボトルネック制約はなし → エラー類型 A/B は該当せず。エラー類型 C（スコア最大化）モードで実行。

ソルバーが解を返した後に HC1-HC8 を独立に全件チェックし、違反ゼロを確認。

### 何が効いたか

- **balanced が最高スコア (90.0)**: 全 SC をバランスよく考慮した重みで、どれも極端には悪化しない
- **home_clinic 偏重は逆効果 (77.6)**: SC4 を 94.4→96.5 に 2 点上げる代わりに、SC2 が 62.5→2.5、SC5 が 83.3→66.7 と大幅悪化
- SC1 (連続勤務) と SC3 (最低時間) は全シナリオで 100 達成 → 構造的に余裕がある
- SC6 (夜勤) も全シナリオで 100 → 余裕ある

（各スコアは 0-100。100 = 完全遵守）

### トレードオフの観察

1. **V3 (home_max) は明確なトレードオフ**:
   - SC4 (home fit): 94.4 → 96.5 (+2.1)
   - **SC2 (公平性): 55.0 → 2.5 (-52.5)** ← 特定の看護師が他クリニックを避けるため、時間配分が偏る
   - SC5 (週末): 83.3 → 66.7 (-16.6)
   - → **home_clinic を優先するほど、他の公平性指標が壊れる**

2. **V1/V4/V5 はほぼ同値**:
   - 重みを変えても同じ解に到達（問題が十分大きく、重みの影響が少ない領域）
   - 構造的な「最良解の集合」に到達していることを示唆

3. **どの方針がベストか**: 顧客の優先度次第
   - 「とにかくバランス」→ V1 (default)
   - 「home_clinic が最優先」→ V3（ただし他の指標とのトレードオフを理解した上で）

## 4. まだ改善余地がある指標

- **SC2 (公平性) = 62.5 in A**: 勤務時間の最大-最小差が 15h（range 50-65h）
  - 原因: 看護師の max_hours が異なる (32h vs 40h) + 不可日の分布
  - 改善案: 時間単位ではなく「稼働率」で公平性を測る（max_hours 比）
  - これは v2 で検討

- **SC5 (週末配置) = 83.3**: 週末担当数の最大-最小差が 1
  - ほぼ許容範囲だが、完全均等 (差=0) は達成していない
