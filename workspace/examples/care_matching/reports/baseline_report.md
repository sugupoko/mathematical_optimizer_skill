# ベースラインレポート — 介護マッチング最適化

## 1. 実行した3手法

| 手法 | 概要 |
|------|------|
| **Random** | 利用者をランダムに並べ、容量内で適当に割当 |
| **Greedy** | 要介護度の高い利用者から順に、互換性スコアが最大のヘルパーを割当 |
| **CP-SAT (基本)** | Google OR-Tools CP-SAT ソルバーで互換性スコア総和を最大化 |

## 2. 結果比較

## 3. 各手法の詳細

### Random
- マッチ自体は全員にできるが、ハードコンストレイント（資格・経験・NG）を無視するため**9件の違反**が発生
- 継続ペアはゼロ維持。偶然の一致に頼るため品質は低い
- **ベースラインとしての役割**: 制約無視時の最低ラインを示す

### Greedy（要介護度優先）
- 要介護5→4→3→...の順に、互換性スコアが最高のヘルパーを割り当て
- HC違反ゼロ。重度利用者を優先的に処理するため、資格・経験の制約を自然に満たす
- 継続維持率100% — R001-C001, R002-C002, R003-C010, R004-C008, R007-C002(別), R009-C008(別), R010-C005がすべて維持
- **課題**: C009が未配置（負荷0）。C001とC005に3名集中

### CP-SAT（基本）
- Greedy とほぼ同等のスコアだが、負荷分散がわずかに良い（std 0.81 vs 0.92）
- C009にR014を配置し、C003を未配置にした点が差分
- **ソルバーは最適解を高速に発見**（問題サイズが小さいため数秒で解ける）

## 4. ボトルネック分析

### ボトルネック1: C002とC008への集中
- C002: R002（満足度5）+ R007（満足度5）の2名。容量4なので問題ないが、世田谷区の3利用者(R002,R007,R012)のうち2名を担当
- C008: R004（満足度5）+ R009（満足度5）の2名。いずれも要介護5で最も重い利用者

これらのペアは信頼関係が深く、変更のリスクが高い。現状の解で継続維持できている。

### ボトルネック2: 性別一致率が80%止まり
- R012(山田久美子)は女性希望だが、世田谷区の女性ヘルパーがいない
- C007(西村隆/男性)が割当されている。利用者が了承済みのため実運用上は問題ないが、数値上は改善余地

### ボトルネック3: ヘルパー負荷の偏り
- Greedy解ではC009（初任者研修/中野区/男性）が未配置
- C009の対応可能曜日がTue,Thu,Satと少なく、かつ経験1年のため要介護4以上に配置不可
- 生活援助のみ・要介護3以下・曜日一致の利用者にしか配置できない

## 5. 改善の方向性

1. **公平性の強化**: 全ヘルパーに最低1名は配置する制約を追加
2. **継続ペアの維持をソフト制約として明示的にモデル化**: 既にGreedyで維持できているが、制約が増えた場合のロバスト性のため
3. **性別一致率の改善**: R012への対応検討（隣接区からの女性ヘルパー配置可能性）
4. **経験10年以上の優先配置**: 要介護5の3名に経験10年以上のヘルパーを優先（SC06）
