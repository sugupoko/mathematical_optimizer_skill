# 改善レポート — 介護マッチング最適化

## 1. 改善手法

CP-SAT ソルバーに以下のソフト制約をボーナスとしてモデル化:

| 要素 | ボーナス値 | 狙い |
|------|----------|------|
| マッチボーナス | +5000/人 | 全員マッチングを最優先 |
| 継続ボーナス（満足度5） | +3000/ペア | 信頼関係のあるペアを維持 |
| 性別一致ボーナス | +800/ペア | 利用者の性別希望を尊重 |
| 同一地区ボーナス | +600/ペア | 移動距離を短縮 |
| 隣接地区ボーナス | +300/ペア | 隣接区も許容 |
| 経験10年以上（要介護5） | +500/ペア | 重度利用者に熟練ヘルパー |
| 最大負荷ペナルティ | -1000×max_load | 負荷集中を抑制 |

## 3. 分析と考察

### なぜスコアが同じか

この問題は15利用者×10ヘルパーと小規模であり、かつ供給が需要の2.67倍ある。Greedy（要介護度優先）が既にほぼ最適解に到達している。具体的には:

1. **満足度5のペア7組のうち、競合するのはC002(R002 vs R007)とC008(R004 vs R009)の2組** — いずれも容量内で両方割当可能
2. **資格制約が厳しい利用者（身体介護・要介護4以上）が自然にフィルタされる** — 貪欲法でも最適に近い割当が見つかる
3. **地区の均等配置** — 各区2名のヘルパーで自然に同一地区マッチングが成立

### 負荷分散について

- CP-SAT基本版は互換性最大化の過程でC009(中野区/初任者研修/経験1年)にR014(中野区/要介護1/生活援助)を配置し、std=0.81を達成
- CP-SAT改善版は継続・性別・地区のボーナスが加わることでR014をC005に配置する解を選び、std=0.92に
- C009を活用するにはR014への配置が最有力だが、C005(中野区/介護福祉士/経験10年)の方が互換性は高い

### 改善が効いた点

改善版は**明示的な目的関数のモデル化によるロバスト性**が主な利点。現在のデータでは差が出にくいが:

- 制約が追加された場合（例: ヘルパーの休暇、新規利用者の追加）にGreedyが崩壊しやすいのに対し、CP-SAT改善版は柔軟に再最適化できる
- ボーナスの重み調整で「公平性重視」「継続性重視」などの方針転換が容易
