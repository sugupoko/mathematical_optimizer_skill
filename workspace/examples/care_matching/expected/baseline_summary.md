# ベースライン結果サマリー — 介護マッチング

## 問題の概要

- 利用者数: 15名（要介護1: 2名、要介護2: 3名、要介護3: 4名、要介護4: 3名、要介護5: 3名）
- ヘルパー数: 10名（介護福祉士: 7名、初任者研修: 3名）
- ハード制約: 5件、ソフト制約: 7件

## ベースライン1: ランダム割当

| 指標 | 結果 |
|------|------|
| マッチ成立数 | 15/15（100%） |
| ハード制約違反 | 4件 |
| 内訳: 資格違反 | 2件（初任者研修のヘルパーが身体介護に配置） |
| 内訳: 経験不足 | 1件（経験1年のヘルパーが要介護4に配置） |
| 内訳: NGペア | 1件（R004-C004の組合せが発生） |
| 性別希望一致率 | 40%（6名中） |
| 同一区内率 | 33% |
| 平均満足度（既知ペア） | 3.2 |
| 担当数の偏り（標準偏差） | 1.8 |

**評価**: ハード制約違反が4件あり実用不可。

## ボトルネック分析

### ボトルネック1: ヘルパーの供給不足

- 身体介護が必要な利用者: 9名（身体介護6名 + both3名）
- 介護福祉士のヘルパー: 7名
- 要介護4以上（経験3年以上必要）: 6名、対応可能ヘルパー: 5名
- **需給がギリギリ**。1名でも欠勤すると制約違反が避けられない。

### ボトルネック2: 地区の偏り

- 杉並区の利用者: 4名、杉並区のヘルパー: 2名（C001, C006）
- C006は初任者研修のため身体介護不可 → 杉並区の身体介護はC001に集中
- 渋谷区も同様の構造（利用者2名、ヘルパー2名だがC010は経験6年）

### ボトルネック3: 性別希望の充足

- 女性ヘルパー希望の利用者: 6名
- 女性ヘルパー: 6名（C001, C003, C005, C006, C008, C010）
- 数は足りるが、地区・資格の制約と組み合わせると選択肢が限られる

## 改善の方向性

1. **目的関数と評価関数の一致**: 満足度だけでなく、公平性（担当数の均等化）と継続性を目的関数に組み込む
2. **ペナルティの重み調整**: 性別希望違反のペナルティ、地区外配置のペナルティの重みを現場の優先度に合わせる
3. **ロバスト性の確保**: ヘルパー1名欠勤時のバックアップ割当を事前に計算しておく
4. **データ追加の検討**: 全ペアの相性予測のため、利用者・ヘルパーの属性ベースの類似度モデルを検討
