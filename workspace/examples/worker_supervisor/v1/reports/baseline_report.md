# Baseline Report — worker_supervisor v1

**日付**: 2026-04-11
**戦略**: 段階的解法 (complexity=complex)
**ソルバー**: OR-Tools CP-SAT, time_limit=60s/phase, num_workers=4

## 概要

20 作業者 + 8 監督者 × 42 シフト (2週間) = **1,176 バイナリ変数** の複合スケジューリング問題。
12 個の HC を 7 Phase で段階的に追加し、どこで壁にぶつかるか特定。

## Phase 別結果

### 読み方

各 Phase でソルバーに渡す制約を段階的に増やす。独立検証器は**全 12 HC** をチェックするが、結果を 2 つに分けて報告:

- **active**: その Phase で**すでに強制している** HC。ソルバーが正しければ違反ゼロになるはず → ゼロでなければ実装バグ
- **pending**: まだモデルに入れていない HC。違反があっても OK (後の Phase で解消する対象)

### Phase 表

| Phase | 追加 | ソルバー | w割当 | v割当 | active HC | pending 違反 (Δ) | 時間 |
|-------|------|:---:|-----:|-----:|:---:|---|---:|
| 0 | - | OPTIMAL | 0 | 0 | ✓ (なし) | 88 | 0.04s |
| 1 | HC1 | OPTIMAL | 160 | 0 | ✓ | 135 (Δ +47) | 0.01s |
| 2 | HC2 | OPTIMAL | 160 | 46 | ✓ | 116 (Δ −19) | 0.01s |
| 3 | HC3, HC4 | OPTIMAL | 160 | 46 | ✓ | 88 (Δ −28) | 0.04s |
| 4 | HC5, HC6 | OPTIMAL | 160 | 46 | ✓ | 70 (Δ −18) | 0.03s |
| **5** | **HC7, HC8** | **INFEASIBLE** | - | - | - | - | 0.01s |
| 6 | HC9, HC10 | INFEASIBLE | - | - | - | - | 0.01s |
| 7 | HC11, HC12 | INFEASIBLE | - | - | - | - | 0.01s |

### 観察

1. **Phase 0-4 は全 active OK**: ソルバーが追加された制約を常に 100% 満たしている (実装バグなし)
2. **Phase 1 → 2 で pending 一時悪化** (+47 から −19 に回復): HC1 だけで解くと HC2-12 方向に副作用が出るが、HC2 追加で戻る (正常な挙動)
3. **Phase 5 で初めて INFEASIBLE**: Phase 4 まで feasible → Phase 5 で HC7, HC8 を追加した瞬間に解が消滅
4. **Phase 6, 7 は Phase 5 を引き継いだだけ** → HC9-HC12 は**無実**

**壁: Phase 5 で追加した HC7 (skill) + HC8 (bilingual) の組合せ**

## 根本原因分析

### なぜ infeasible か、数値で示す

データから候補・需要を計数:

**候補プール (en を話せる作業者でかつ reception スキルを持つ者):**

| 作業者ID | 名前 | 備考 |
|---|---|---|
| W002 | 佐藤花子 | en + reception |
| W004 | 高橋美咲 | en + reception |
| W008 | 中村さくら | en + reception |
| W010 | 加藤真理 | en + reception |

→ 合計 **4 名**

**該当する需要 (bilingual_required=yes かつ required_skills=reception):**

| シフト | 必要人数 |
|---|---:|
| 第1週 火曜 morning | 5 |
| 第1週 木曜 morning | 5 |
| 第2週 火曜 morning | 5 |
| 第2週 木曜 morning | 5 |

→ 各シフト **5 名必要**

### 鳩の巣原理による不可能性

```
任意の bilingual+reception シフト s について:
  HC1 (需要): Σ_w wx[w, s] = 5
  HC7 (skill): wx[w, s] = 0  (w が reception を持たない)
  HC8 (lang): wx[w, s] = 0  (w が en を話せない)

→ 候補は {W002, W004, W008, W010} の 4 名のみ
→ Σ ≤ 4 < 5  ⇒ 矛盾
```

これは**プールサイズ < 必要人数**の典型的な構造的不可能性であり、
どんなアルゴリズムを使っても、どんなソルバーを使っても、どんな重み調整をしても絶対に解けない。

### 補助的観察

- **bilingual + phone** のシフト (Thu/Fri afternoon × 2週) も存在し、各 5 名要求
- en + phone の候補は 6 名 (W002, W004, W008, W010, W013, W017)
- 6 ≥ 5 なので 1 シフト単体では満たせるが、2 週で延べ 20 worker-shifts 必要 → タイト
- 今回の壁の主因は **reception 側の 4 名ギャップ** (こちらが先に破綻)

### 無実の制約 (容疑から除外)

段階化により以下が無実と判明:

- **HC9, HC10 (休息制約)**: Phase 5 で既に壁のため効果を評価できない → **無罪** (Phase 6 が独自に壁になるには Phase 5 が feasible である必要がある)
- **HC11 (forbidden ペア)**: 2 件のみ、Phase 5 の壁と無関係
- **HC12 (mentorship ペア)**: 4 件、2 週間で同一シフト 2 回達成は鳩の巣と無関係

→ **ボトルネックは純粋に HC1 ∩ HC7 ∩ HC8 の組合せ**。他の複雑な制約は原因ではない。

## 解決策の選択肢

### A. 入力を変える (データ側)

| 案 | 内容 | コスト | 導入難易度 |
|---|---|---|---|
| **A1. 英語可+reception 作業者を 1 名追加採用** | W021 (en + reception + phone/chat) を雇う | 人件費 (中) | 1-2 ヶ月 |
| A2. 既存の英語可作業者に reception 研修 | W013 or W017 に reception スキル習得させる | 研修費 | 2-3 ヶ月 |
| A3. 既存 reception 作業者に英語研修 | 非英語の reception 保持者に ESL 研修 | 研修費 + 時間 | 6 ヶ月以上 |

### B. 仕様を変える (HC の緩和)

| 案 | 内容 | コスト | 導入難易度 |
|---|---|---|---|
| **B1. HC1 緩和**: bilingual+reception シフトを 5 → 4 名 | 英語顧客対応キャパが 20% 減 | 0 円 | 顧客合意が必要 |
| B2. HC8 の解釈変更: 「全員 en 必須」→「最低 1 名 en」 | spec の意味を緩める | 0 円 | **クライアント確認必須 (Q5)** |
| B3. HC1 を soft 化 (shortage を許容) | penalty 付きで shortage 許容 | 0 円 | サービス品質合意 |

### C. 運用で回避 (プロセス変更)

| 案 | 内容 | コスト | 導入難易度 |
|---|---|---|---|
| C1. bilingual 需要の時間帯再配置 | 英語顧客対応を別時間帯に分散 | 顧客導線変更 | 顧客との調整 |
| C2. 一時派遣で補う | 4 シフトだけ外部スタッフで | 外注費 | 契約締結のみ |

### 推奨組合せ

1. **即時**: spec の **Q5 を確認** (B2 が成立すればコストゼロで解決)
2. **Q5 が A (全員必須) だった場合**: B1 (HC1 緩和) で暫定運用
3. **中期**: A1 (W021 採用) を恒久対策として進める

## 段階化の価値

**もし一発解きなら**: 12 個の HC を全部入れて INFEASIBLE と返るだけで、どの制約が原因か特定できない。HC9-12 (休息・ペア) を疑って時間を浪費するリスクあり。

**段階化により判明したこと**:
1. HC1-HC6 (基礎需要・供給・可用性) は **完全に通る** (160/160, 46/46)
2. Phase 5 で HC7+HC8 を追加した瞬間に壁
3. 原因は **スキル + 言語の組合せ問題のみ**
4. HC9-HC12 は **無実**
5. 切り分け時間: 全 Phase 合計 **約 0.15 秒** (Phase 0 の 0.04s を含む)
