# ベースラインレポート

実施日: 2026-04-07
対象: シフトスケジューリング最適化（10名×7日×3シフト）

## 1. 実験設定

3つのベースライン手法で解を生成し、全制約（HC1-5, SC1-5）に対する違反数とスコアを評価した。

| 手法 | 概要 | 実行時間 |
|------|------|---------|
| ランダム | 各シフトにランダムに従業員を割当（max_hours制約のみ考慮） | <0.01s |
| 貪欲法 | スキル・不可日を考慮し、余力が大きい従業員から優先割当 | <0.01s |
| CP-SAT | OR-Tools CP-SATで全HC+SCを最適化 | 0.06s |

## 3. ボトルネック分析

### 3.1 構造的人員不足（最重要）

**総需要48シフト（384h）に対し、総供給上限46シフト（368h）。最低2シフトが埋まらない。**

これはソルバーの性能とは無関係の構造的制約であり、以下のいずれかでしか解消できない:
- 人員の追加（シナリオA）
- 夜勤の最低人数緩和（シナリオB）
- シフト需要の削減（金曜午後3→2など）

### 3.2 CP-SATの未充足シフト

CP-SATが埋められなかったシフト:
- **水曜 朝勤**: 必要3名に対し2名（1名不足）
- **日曜 朝勤**: 必要2名に対し1名（1名不足）

理由: 朝勤はreceptionスキルが必要（8名対応可能）だが、不可日・HC5（夜勤翌日朝禁止）・max_hours制約の複合で、この2枠が最後に溢れる。

### 3.3 公平性の限界（SC2 = 30.1）

SC2が低い原因は、従業員間のmax_hours格差:
- E008: max 24h（3シフト分のみ）
- E003, E006: max 32h（4シフト分）
- その他: max 40h（5シフト分）

供給不足のため、max 40hの従業員はフル稼働が必要。結果として時間の偏り（std_dev=5.59h）は構造的に避けられない。

### 3.4 手法間比較

- **ランダム→貪欲法**: HC3, HC4を完全解消（スキル・不可日の考慮で-10違反）
- **貪欲法→CP-SAT**: HC5を完全解消（夜→朝の順序制約で-2違反）
- **CP-SATのHC1=2は構造限界**であり、これ以上の改善はモデル変更が必要

## 4. 次のステップ

1. **シナリオA（1名追加）**: HC1=0を達成可能か検証
2. **シナリオB（夜勤緩和）**: 夜勤1名でHC1=0を達成可能か検証
3. **シナリオC（公平性重視）**: 重み調整でSC2改善の余地を探る
4. **不可能性の証明**: 現行10名体制でのHC1=0不可能を数理的に示す
