# アセスメントレポート: シフト最適化

## 1. 問題の概要

- **従業員数**: 10名
- **対象期間**: 1週間 (月〜日)
- **シフト種別**: 3種 (morning 09:00-17:00, afternoon 13:00-21:00, night 21:00-05:00)
- **シフト枠数**: 21枠/週 (平日 各3枠, 土日 各3枠, 金曜午後は3人必要)
- **全シフト8時間勤務**

## 2. 供給 vs 需要分析

### 総量

| 指標 | 値 |
|------|-----|
| 総需要スロット | 48人・シフト |
| 総需要時間 | 384時間 |
| 総供給時間(上限) | 368時間 |
| 供給/需要比 | **0.96 (供給不足)** |

**結論: 全シフトの必要人数を完全に満たすことは構造的に不可能。最低2スロットの人員不足が発生する。**

### 日別分析

| 曜日 | 需要(枠) | 利用可能(人) | 状態 |
|------|----------|-------------|------|
| Mon | 7 | 10 | OK |
| Tue | 7 | 10 | OK |
| Wed | 7 | 9 | OK |
| Thu | 7 | 10 | OK |
| Fri | 8 | 10 | OK |
| Sat | 6 | 8 | OK |
| Sun | 6 | 8 | OK |

日単位では余裕があるように見えるが、従業員ごとの週最大勤務時間制約（HC2）とスキル制約（HC4）を加味すると供給不足になる。

### スキル別分析

| スキル | 需要(延べ) | 保有者数 |
|--------|----------|---------|
| reception | 19枠 | 8名 |
| phone | 29枠 | 10名 |
| training | (各日1名以上) | 5名 |

- phoneスキルは全員保有
- receptionは8名が保有（E005, E009は不保有）
- trainingスキル保有者: E001, E003, E005, E007, E010 (5名)

## 3. 制約の整理

### ハード制約 (必ず守る)

| ID | 内容 | リスク |
|----|------|--------|
| HC1 | 必要人数充足 | **構造的に不可能** (供給368h < 需要384h) |
| HC2 | 週最大勤務時間 | E003, E006は32h上限、E008は24h上限 |
| HC3 | 利用不可日 | E002(日), E004(土), E006(土日), E008(水) |
| HC4 | スキル要件 | receptionはE005, E009に割当不可 |
| HC5 | 夜勤→翌日朝勤禁止 | 11時間休息ルール |

### ソフト制約 (できるだけ守る)

| ID | 内容 | 重み |
|----|------|------|
| SC1 | 連続勤務5日以下 | 10 |
| SC2 | 勤務時間の公平性 | 5 |
| SC3 | 最低勤務時間確保 | 3 |
| SC4 | 夜勤週2回以下 | 8 |
| SC5 | トレーニング持ち毎日配置 | 5 |

## 4. 仮説

1. **HC1は完全充足不可能** → 2スロットの不足が理論下限
2. **夜勤が最大のボトルネック** → 週14枠の夜勤にphone持ち10名で回す必要があり、HC2・HC5制約と競合
3. **E008 (24h上限) が供給の弱点** → 最大3シフト/週しか入れない
4. **土日のE006不在が土日の供給を圧迫** → 利用可能8名で6枠を埋める

## 5. 推奨アプローチ

1. HC1を緩和（ソフト化）し、不足を最小化する定式化を使用
2. 不足が出る場合、影響が最も小さいシフト（夜勤など）に集中させる
3. 経営判断として「夜勤の人員削減」または「パート増員」を提案
