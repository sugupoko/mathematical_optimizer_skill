# シフト最適化 改善提案書 (v1)

**作成日**: 2026-03-30
**対象**: 10名体制のシフトスケジューリング（1週間）

---

## エグゼクティブサマリー

現在のシフト体制（10名、週48枠）を数理最適化で分析した結果、**構造的な人員不足**が判明しました。現行の必要人数を完全に満たすには、週384時間の労働力が必要ですが、現在の10名体制では最大368時間しか確保できません。

最適化ソルバーにより、**不足を最小限（2枠）に抑えつつ、夜勤の偏りを解消し、全従業員の最低勤務時間を確保する**シフト表を生成しました。

---

## 1. 現状の課題

### 人員不足の構造

```
必要労働時間: 384時間/週
供給可能時間: 368時間/週 (上限)
不足:          16時間 = 2シフト分
```

この不足は「誰かがもっと働けば解決する」問題ではなく、全員が上限まで働いても16時間足りない**構造的な問題**です。

### 制約の厳しさ

- E008（中村さくら）: 週24時間上限 → 最大3シフト
- E003（鈴木一郎）, E006（渡辺由美）: 週32時間上限 → 最大4シフト
- E006: 土日出勤不可 → 土日の供給がさらに制約
- E002: 日曜出勤不可、E004: 土曜出勤不可

---

## 2. 最適化の結果

5つの手法を比較し、最良の手法を特定しました。

| 手法 | 制約違反数 | 品質スコア | 勤務偏差 |
|------|----------|----------|---------|
| ランダム割当 | 8件 | -62.29 | 15.46h |
| 貪欲法 | 3件 | -14.99 | 6.80h |
| ソルバー基本版 | 2件 | -8.97 | 5.59h |
| **ソルバー改善版** | **2件** | **+7.03** | **5.59h** |
| 2フェーズ法 | 2件 | +7.03 | 5.59h |

**最良手法（ソルバー改善版）の特徴**:
- 人員不足を理論上の最小値（2枠）に抑制
- 夜勤を全従業員で週2回以下に均等化（従来は偏りあり）
- 全従業員の最低勤務時間を確保
- 毎日トレーニングスキル保持者を配置

---

## 3. 提案

### 提案A: 現行10名体制での最適シフト（即時実施可能）

ソルバー改善版で生成したシフト表を採用する。

**メリット**:
- 追加コストなし
- 夜勤の公平性が大幅改善
- 制約違反は最小限（2枠の人員不足のみ）

**デメリット**:
- 2枠の人員不足は解消できない
- 全従業員が上限まで勤務（余裕なし）

### 提案B: 夜勤の必要人数を調整（運用変更）

夜勤の必要人数を一部の曜日で2名→1名に削減する。

**例**: 土曜・日曜の夜勤を1名体制にすると、週需要が44枠（352時間）となり、供給368時間で余裕が生まれる。

**メリット**:
- 全シフト充足可能に
- 従業員に余裕が生まれ、急な欠勤にも対応可能

**デメリット**:
- 夜勤1名体制のリスク（安全面の確認が必要）

### 提案C: パート従業員の追加（中長期）

週16-24時間勤務のパート従業員を1名追加する。

**メリット**:
- 構造的な供給不足を根本解決
- 既存従業員の負担軽減
- 急な欠勤への対応力向上

**コスト見積もり**: 週16時間 x 時給 x 52週/年

---

## 4. 推奨

**短期（今週から）**: 提案Aを即時採用。ソルバー生成のシフト表で運用開始。

**中期（1-2ヶ月）**: 提案Bの夜勤体制見直しを検討。土日夜勤1名体制の安全性を評価。

**長期（3ヶ月以降）**: 提案Cのパート採用を検討。需要が増加傾向ならば、先手を打つべき。

## 5. 補足: 技術的詳細

- **使用ツール**: Google OR-Tools CP-SAT ソルバー
- **最適化手法**: 混合整数計画法 + ソフト制約の目的関数組み込み
- **計算時間**: 約2分（120秒タイムリミット）
- **最適性**: OPTIMAL（証明付き最適解）

### 供給不足の数学的証明

```
従業員の週最大勤務時間合計:
  E001(40) + E002(40) + E003(32) + E004(40) + E005(40)
+ E006(32) + E007(40) + E008(24) + E009(40) + E010(40) = 368時間

シフト必要時間:
  平日5日 x 7枠 x 8h + 金曜追加1枠 x 8h + 土日2日 x 6枠 x 8h
  = 280 + 8 + 96 = 384時間

不足: 384 - 368 = 16時間 = 2シフト分
```

この不足は、利用不可日やスキル制約を考慮する前の理論下限である。実際にはこれらの制約によりさらに厳しくなるが、最適化ソルバーにより理論下限の2枠で抑えることに成功した。
