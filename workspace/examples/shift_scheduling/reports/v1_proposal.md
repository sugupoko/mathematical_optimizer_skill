# シフト最適化 — 改善提案書

**対象**: 10名体制 × 7日間 × 3交代シフト
**作成日**: 2026-03-30
**ステータス**: 分析完了、提案3案

---

## 1. 現状の課題

### 人員が足りていません

現在の勤務条件（週最大時間・休日・スキル要件）を全て守ると、**必要な48シフト枠に対して最大46名分しか配置できません**。

これはアルゴリズムの問題ではなく、**人員と需要のミスマッチ**です。どんな手法を使っても、現在の条件では全シフトを埋めることは数学的に不可能です。

| 指標 | 数値 |
|------|------|
| 週間の必要シフト枠 | 48枠 |
| 配置可能な最大シフト数 | **46枠** |
| 不足 | **2枠** |
| 不足箇所 | 水曜朝勤（3名中1名不足）、日曜午後勤（2名中1名不足） |

### 不足の原因

```
供給の内訳:
  10名の従業員 × 各自の上限（3〜5シフト/週） = 最大46シフト/週

  特にタイトな従業員:
    中村 さくら: 週3シフトが上限（24h制限）、水曜不可
    渡辺 由美:   週4シフトが上限（32h制限）、土日不可
    鈴木 一郎:   週4シフトが上限（32h制限）

  特にタイトな曜日:
    水曜: 中村さくら不可 → reception可能者が7名に減少
    日曜: 佐藤花子・渡辺由美不可 → 利用可能者が8名に減少
```

### 手作業（貪欲法）で作ったシフト表の問題

ルールを意識して手で組んでも **5枠が埋まらない** 状態が発生しました（実験で確認）。「なんとなく組める気がする」が実際には組めないケースです。

### 提案A: 需要の調整（推奨）

**水曜の朝勤を3名→2名、日曜の午後勤を2名→1名に変更する。**

この2箇所はもともと他の曜日より業務量が少ない（水曜は中日、日曜午後は来客が少ない）可能性が高く、運用上の影響は最小限です。

**効果:**
- 全21シフト × 7日 = **全枠が埋まる**（ハード制約違反ゼロ）
- ソフト制約も高水準で達成:

| ソフト制約 | 達成状況 |
|-----------|---------|
| 連続勤務5日以下 | **達成** |
| 最低勤務時間の確保 | **全員達成** |
| 夜勤は週2回以下 | **達成** |
| トレーニング有資格者の毎日配置 | **達成** |
| 勤務時間の公平性 | 24h〜40h（標準偏差5.3h） |

**勤務時間の配分:**

| 従業員 | 勤務時間 | 備考 |
|--------|---------|------|
| 田中 太郎 | 40h | 上限 |
| 佐藤 花子 | 40h | 上限（日曜休み） |
| 鈴木 一郎 | 32h | 上限 |
| 高橋 美咲 | 40h | 上限（土曜休み） |
| 伊藤 健太 | 40h | 上限 |
| 渡辺 由美 | 32h | 上限（土日休み） |
| 山本 大輔 | 40h | 上限 |
| 中村 さくら | 24h | 上限（水曜休み） |
| 小林 裕二 | 40h | 上限 |
| 加藤 真理 | 40h | 上限 |

ほぼ全員が上限まで働く形になります。これは供給46 ≒ 需要46 でギリギリのため必然です。

### 提案B: パートタイム従業員の追加

水曜・日曜に週2回出勤できるパートを1名雇用すれば、元の需要48枠を維持できます。

**必要なスキル:** reception（水曜朝勤用） + phone（日曜午後用）
**必要な勤務時間:** 週16h（2シフト × 8h）

### 提案C: 現状のまま運用（非推奨）

2枠が常に人手不足の状態を許容する。

- 水曜朝勤: 3名必要なところ2名で対応
- 日曜午後勤: 2名必要なところ1名で対応
- **リスク**: サービス品質低下、残った従業員への負荷集中

## 3. 数学的な限界（不可能性の証明）

以下は数学的に証明済みです:

> **現在の10名体制・勤務条件のもとでは、週48シフト枠を全て埋めることは不可能。**
> 最大でも46枠しか埋まらない。

証明方法:
- CP-SATソルバーで最大割当数を計算 → **46が上限**
- HC5（夜勤→朝勤禁止）を撤廃しても上限は変わらない → ボトルネックは人員の絶対数
- 需要を1枠だけ削減（47枠）でもまだ不可能 → **最低2枠の削減が必要**

この結果は「もっと良いアルゴリズム」では解決できません。問題の条件そのものを変える必要があります。

## 4. 推奨アクション

### すぐやること
1. **水曜朝勤と日曜午後勤の必要人数を確認** — 本当に3名/2名必要か？減らせないか？
2. 減らせるなら **提案Aを即日適用** — ソルバーで0.02秒でシフト表が生成される

### 中期（1〜2ヶ月）
3. 需要を減らせない場合、**パートタイム1名の採用を検討**（提案B）

### 継続的な改善
4. 繁忙期・閑散期のデータを収集し、曜日別の必要人数を最適化
5. 従業員の希望シフトデータを追加すれば、公平性スコアをさらに改善可能
