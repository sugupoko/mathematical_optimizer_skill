# 改善レポート: 配送ルート最適化

## 1. 改善概要

ベースライン（OR-Toolsソルバー1便、19/20顧客配送）から、AM/PM分割による2便運行を実施した。

**目的**: 容量不足（815kg > 750kg）を2便運行で解消し、全20顧客への配送を実現する。

## 2. 改善手法: AM/PM分割

### アプローチ

V003（200kg、稼働6時間）を午前便・午後便の2便に分割運行する。

```
AM便: V001 + V002 + V003（午前） → デポ帰還
PM便: V003（午後）               → デポ帰還
```

V001とV002は1便で完了し、V003のみがデポに戻って再出発する。これにより実質的な総容量が増加する。

### 顧客の時間帯割り当て

- **午前配送（AM便）**: 時間枠が早い顧客を優先。3台で16顧客を配送
- **午後配送（PM便）**: V003が再出発し、午後時間枠の残り4顧客を配送

## 5. コスト増分の分析

| 項目 | 値 |

| 距離増分 | +21.29 km/日（デポ往復1回分） |

このコスト増は、V003のデポ往復（午前帰還+午後出発）による追加走行が主因。V001とV002は1便で完了しており、追加走行はない。

## 6. 改善のポイント

### うまくいったこと

1. **全20顧客への配送を達成**: 容量不足の構造的問題をAM/PM分割で解消
2. **時間枠違反ゼロを維持**: 午前便で早朝顧客、午後便で午後顧客を効率的に割り当て
3. **V001/V002の積載効率が高い**: 98.3%、96.0%と無駄なく活用
4. **V003の午前便は軽量**: 2顧客のみで早期帰還し、午後便の余裕を確保

### 残存課題

1. **走行距離の増加（+25.2%）**: デポ往復の追加コストは避けられない
2. **V003ドライバーの負荷**: 2便運行になるため、休憩時間の確保が必要
3. **渋滞の影響が未考慮**: 午後便の出発時刻が遅れるリスクがある
4. **仮定の検証が必要**: 移動速度30km/h、道路係数1.3は実績データで要確認
