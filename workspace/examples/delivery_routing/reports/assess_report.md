# データ分析レポート（アセスメント）: 配送ルート最適化

## 1. 問題概要

| 項目 | 値 |
|------|-----|
| デポ | 品川配送センター |
| 顧客数 | 20（東京都内） |
| 車両数 | 3台 |
| 総需要量 | 815 kg |
| 総積載容量（1便） | 750 kg |
| 容量比率 | 1.087（**容量不足**） |

## 2. 問題分類

- **問題タイプ**: 容量制約付き配送経路問題（CVRPTW: Capacitated VRP with Time Windows）
- **主要制約**: 積載容量、時間枠、稼働時間上限
- **最大の課題**: 総需要 815 kg > 総容量 750 kg（65 kg超過）
- **問題規模**: 小規模（20地点、組合せ空間は限定的）

## 3. データ特性

### 3.1 距離

| 指標 | 値 |
|------|-----|
| デポからの平均距離 | 7.99 km |
| 顧客間の平均距離 | 7.96 km |
| デポからの最大距離 | 14.97 km（錦糸町店 C020） |

東京都心に分布しており、移動距離は短い。地理的クラスタリングによるルート効率化の余地がある。

### 3.2 時間枠

| 指標 | 値 |
|------|-----|
| 平均時間枠幅 | 216分（3時間36分） |
| 最小時間枠幅 | 120分（2時間） |
| 時間枠最狭の顧客 | C010 大崎倉庫（08:00-10:00）、C004 上野商店（09:00-11:00） |

時間枠は比較的余裕がある。ただし早朝限定の顧客（C010, C006）はルート構成上の制約になる。

### 3.3 車両スペック

| 車両ID | 積載量(kg) | 最大稼働(h) | コスト(円/km) | 備考 |
|--------|-----------|------------|--------------|------|
| V001 | 300 | 8.0 | 120 | 大型・高コスト |
| V002 | 250 | 8.0 | 110 | 中型 |
| V003 | 200 | 6.0 | 100 | 小型・稼働時間短い |

### 3.4 需要分布

| 需要帯 | 顧客数 | 顧客例 |
|--------|--------|--------|
| 大口（60kg以上） | 4 | C018 お台場モール(80kg), C010 大崎倉庫(70kg), C019 豊洲センター(65kg), C003 池袋センター(60kg) |
| 中口（30-55kg） | 10 | C014 秋葉原電気街(55kg), C006 東京駅前店(50kg) 他 |
| 小口（25kg以下） | 6 | C007 六本木オフィス(20kg), C012 恵比寿カフェ(15kg) 他 |

大口4顧客だけで275kg（総需要の33.7%）を占める。

## 4. 容量分析

```
総需要:   815 kg
総容量:   750 kg（V001:300 + V002:250 + V003:200）
超過:     65 kg
容量比率: 1.087（8.7%超過）
```

**1便運行では全20顧客への配送は物理的に不可能。**

最大需要の顧客 C018（お台場モール: 80kg）を外しても 735kg であり、残り3台の容量ギリギリ。現実的には1顧客以上の未配送が発生する。

## 5. 仮説

1. **容量不足が最大のボトルネック**: 1便では全顧客に配送不可能。AM/PM分割による2便運行が必要
2. **時間枠は比較的緩い**: 平均3.6時間の幅があり、ルート最適化の自由度は高い
3. **V003の制約が厳しい**: 容量200kg・稼働6時間と小さく、担当可能な顧客数が限られる
4. **エリア集約が効果的**: 都心部に集中しており、地理的クラスタリングで走行距離を削減可能
5. **AM/PM分割で全顧客カバー可能**: 各便の需要を容量内に収めれば制約充足は容易

## 6. 不足情報

| 項目 | 影響 | 現在の仮定 |
|------|------|----------|
| 渋滞による移動時間変動 | 時間枠違反リスク | 一律30km/h |
| 荷物の体積 | 体積制約の有無 | 重量のみ考慮 |
| 曜日による需要変動 | 車両台数の過不足 | 全日同一需要 |
| 不在時の再配達ルール | コスト増の可能性 | 不在なし |
| 実際のサービス時間 | ルート所要時間精度 | データ記載値を使用 |
