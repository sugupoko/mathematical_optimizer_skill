# 配送ルート最適化 提案書 v1

## エグゼクティブサマリー

品川配送センターから都内20顧客への配送ルートを最適化しました。
**現状の車両3台（総積載750kg）では総需要815kgを1便で配送できない**ことが判明し、
AM/PM 2便制の導入により**全顧客への配送を達成**しました。

## 1. 現状の課題

### 容量不足（最大のボトルネック）

| 指標 | 数値 |
|------|------|
| 総需要 | **815 kg** |
| 3台合計容量（1便） | **750 kg** |
| 不足 | **65 kg** |

3台を1便で運用すると、OR-Toolsソルバーで120秒探索しても**最大19顧客しかカバーできません**。
これはルート設計の問題ではなく、容量の物理的な限界です。

### 時間枠の制約

以下の顧客は時間枠がタイト（120-150分）:
- **C010 大崎倉庫**: 08:00-10:00（120分）
- **C004 上野商店**: 09:00-11:00（120分）
- **C006 東京駅前店**: 08:30-11:00（150分）

## 3. 提案: AM/PM 2便制（推奨）

### 概要

時間枠の中間値で顧客をAM/PMに分類し、各便を独立にOR-Toolsで最適化。

- **AM便（08:00出発）**: 16顧客、需要635kg
- **PM便（12:30出発）**: 4顧客、需要180kg

### 効果

| 指標 | 1便運用（最善） | AM/PM分割 |
|------|---------------|----------|
| 顧客カバー | 19/20（1社欠損） | **20/20（全社配送）** |
| 時間枠違反 | 0件 | 0件 |
| 容量違反 | 0件 | 0件 |
| 総走行距離 | 84.5 km | 105.8 km（+25%） |
| 総コスト | 9,508円 | 11,709円（+2,201円） |
| Feasible | **No** | **Yes** |

### コスト増分の分析

- 距離増: +21.3 km/日（デポ往復が増えるため）
- コスト増: **+2,201円/日**（月20日稼働で約**44,000円/月**）
- 未配送1社の売上損失・信頼喪失と比較して判断

## 4. 代替案

| 提案 | コスト | 効果 | 導入難易度 |

## 6. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 朝の渋滞 | AM便の遅延 | GPSログによる実走時間の取得 |
| 需要増加 | 再び容量不足 | 4台目の車両追加を検討 |
| PM便の非効率 | 車両稼働率低下 | PM便は必要車両のみ出動 |

## 7. 推奨アクション
