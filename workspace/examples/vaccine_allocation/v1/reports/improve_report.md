## 4. 発見と解釈

- **balanced と min_waste は同じ解に収束** (17,700 人、G1+G2 を完全カバー + G3 に 3,200 人)。現在の容量構造では廃棄削減の余地が少なく、SC4 の重みを変えても解は動かない
- **elderly_first は 3,000 人少ない** (14,700): 優先度の重みが重いと G1 を早めに完全完了させるため、後半の週に G2 を完全カバーする時間がなくなる
- **max_coverage は人数こそ同じ 17,700 だが配分が全く異なる**: G1/G2 を完全にスキップし、G3-G5 に薄く分散。これは「人数だけ」の目的関数の退化した最適解で、現実には採用不可
- **共通の発見**: カバレッジ上限は構造的に **17,700 人 (16.3%)** で確定。容量制約を増やさない限りこれ以上は行けない

## 5. トレードオフの構造

**この問題は「誰に打つか」を最適化するだけで、「何人打てるか」は既に決まっている**:

```
容量上限 46,000 slot/10週
  ↓ HC6/HC7 の 2 回接種連動で ÷ 2
23,000 人 (理論)
  ↓ HC4 (超低温) と HC10 (ホライゾン) で -5,300
17,700 人 (実効)
```

優先度の重みを変えても上限は動かない。**戦略判断は質の問題 (誰) であって量ではない**。

## 7. 依頼者向け解釈 (健康福祉部長向け)

- **ハード上限は 17,700 人 (人口の 16.3%)**。ソフト制約の重みをいくら変えても超えられない
- **上限を引き上げるには物理投資が必要**: 超低温冷凍庫 (S3/S4)、スタッフ増員、2 回目同一会場ルール緩和
- **シナリオ選択は「誰を」の問題**: 優先度重視で G1+G2 を完全カバーする (推奨) か、薄く広く配分するか
- **廃棄は戦略ではなく構造**: VP の余剰は超低温設備がボトルネック。ワクチン種の配分 (VP を減らし VM を増やす) は調達側の政治判断
