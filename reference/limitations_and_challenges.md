# スキルパックの限界と今後のチャレンジ

このスキルパックは **CP-SAT + Python スクリプト + 段階的 baseline** という共通パターンで 14 サンプルを構築してきました。このドキュメントは、**この手法が苦しむ領域** と **今後取り組むべき課題** を整理したものです。

---

## 現時点の守備範囲 (14 サンプル)

- **中規模** (変数 ~10,000 以下)
- **整数/バイナリ決定変数** が主
- **静的・決定論的** な一括計画
- **線形 / 単純非線形** 制約
- CP-SAT が得意とする組合せ最適化

実務問題の 80-90% はこの領域に収まるため、実用性は高い。

## 苦手な 6 カテゴリ

### 🔴 A. 巨大スケール (100k+ 変数)

| 問題 | なぜ詰まるか | 代替アプローチ |
|---|---|---|
| 航空乗務員ロスタリング (数百万ペアリング) | 列生成必須 | Gurobi + Column Generation |
| 全国物流ネットワーク (1000+ 拠点 × 1000+ 車両) | 変数爆発 | Benders 分解、クラスタリング |
| 発電機コミットメント (100 機 × 8760h) | 時間軸が長い | Lagrangian relaxation |

| 半導体工場スケジューリング (数千ジョブ) | FJSP 級で規模大 | 商用 + ヒューリスティック |

**CP-SAT の実用上限**: 快適 ~10k、頑張って ~50k、100k+ は別手法が必要。
**対応策**: 問題分解、列生成、商用ソルバー、専用アルゴリズム。

## 最後に: 選び方の指針

実務問題に直面したとき、以下を確認:

1. **変数数は?** — 10k 以下なら CP-SAT で十分
2. **決定変数は整数/バイナリ?** — はい → CP-SAT、いいえ → scipy/cvxpy
3. **目的関数は線形?** — はい → CP-SAT/MIP、二次 → QP、非線形 → NLP
4. **不確実性はある?** — ない → 決定論、ある → SP/RO
5. **応答時間は?** — 秒〜分 OK → CP-SAT、ms → 専用エンジン
6. **1 人の意思決定?** — はい → 最適化、複数 → ゲーム理論

**この 6 つで半分以上が「はい/OK」ならこのスキルパックで対応可能**。
そうでなければ、上記チャレンジカテゴリを参照して別手法を検討してください。
