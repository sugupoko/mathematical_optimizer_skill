# 仕様書テンプレート（spec.md）

各バージョンフォルダ内に置く。**最新バージョンの spec.md が「今の正」。**
過去バージョンの spec.md と比較すれば、何が変わったかがわかる。

---

以下を `workspace/<project>/v1/spec.md` にコピーして使う。
バージョンが上がったら `v2/spec.md` に複製して更新する。

```markdown
# 仕様書

バージョン: v1
最終更新: YYYY-MM-DD

## 目的

- 主目的: [例: 配送コスト最小化]
- 副目的: [例: ドライバー間の公平性]
- 優先順位: [主目的 > 副目的 / 重み付け（コスト70%、公平性30%）]

## ハード制約（必ず守る）

| # | 制約 | 根拠 | 備考 |
|---|------|------|------|
| H1 | [例: 車両積載量 1000kg 以下] | 物理的制約 | |
| H2 | [例: 営業時間 8:00-18:00] | 就業規則 | |

## ソフト制約（できれば守りたい）

| # | 制約 | 重み | 備考 |
|---|------|------|------|
| S1 | [例: 連続稼働 8h 以内] | 高 | |
| S2 | [例: 同一エリアは同一ドライバー] | 中 | |

## 仮定（データがないため仮の値を置いたもの）

| # | 仮定 | 値 | 根拠 | 検証状況 |
|---|------|-----|------|---------|
| A1 | [例: 平均移動速度] | 30km/h | 一般的な市街地速度 | 未検証 |
| A2 | [例: 1件あたり作業時間] | 10分 | ヒアリング | 確認済 |

## データ

| ファイル | 内容 | 備考 |
|---------|------|------|
| [例: orders.csv] | 注文データ（N件） | |
| [例: vehicles.csv] | 車両マスタ（M台） | |

## 受け入れ要件

### 現場の管理方法
- 現在のツール: [例: Excel / 紙 / ホワイトボード / 専用ソフト]
- 操作する人: [例: 管理者のみ / 現場の担当者も]
- 今のツールの良い点: [例: 自由に書き換えられる]
- 今のツールの不満: [例: 作成に3時間かかる、ミスが多い]

### 結果の表示
- 希望する形式: [例: ガントチャート / 地図 / 一覧表 / 既存Excelの形式]
- 閲覧環境: [例: PC / スマホ / 印刷]

### 手修正
- ソルバー出力の手修正: [あり / なし]
- 修正の頻度・理由: [例: 急な欠勤で週1-2回差し替え]
- 修正後の制約チェック: [必要 / 不要]

### 承認・配布フロー
- 承認者: [例: 課長が確認 → 全員に配布]
- 配布方法: [例: メール / 掲示 / システム登録]

### 更新頻度
- 作成頻度: [例: 週1 / 毎日 / 月1]
- 急な変更への対応: [例: 当日の欠勤、急ぎの注文]

### 納品形態
- [例: Excel書き戻し / PDF / Webアプリ / 既存システム連携（API）]

## 前バージョンからの変更点

（v1 では省略。v2 以降で記載する）

| 項目 | 変更前 | 変更後 | 理由 |
|------|-------|-------|------|
| [例: A1 移動速度] | 30km/h | 22km/h | GPSログ実測値 |
```
