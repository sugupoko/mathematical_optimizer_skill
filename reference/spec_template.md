# 仕様書テンプレート（spec.md）

各バージョンフォルダ内に置く。**最新バージョンの spec.md が「今の正」。**
過去バージョンの spec.md と比較すれば、何が変わったかがわかる。

**全ての制約・仮定に情報源の番号（Ref）を付けること。** 後から「この制約はどこから来たのか」を追跡できるようにする。

---

以下を `workspace/<project>/v1/spec.md` にコピーして使う。
バージョンが上がったら `v2/spec.md` に複製して更新する。

```markdown
# 仕様書

バージョン: v1
最終更新: YYYY-MM-DD

## 情報源

全ての制約・仮定の根拠を追跡するための一覧。各項目の Ref 列がここに紐づく。

| Ref | 種類 | 文書・情報源 | 取得日 | 備考 |
|-----|------|------------|--------|------|
| R1 | データ | employees.csv | YYYY-MM-DD | クライアントから受領 |
| R2 | データ | shifts.csv | YYYY-MM-DD | クライアントから受領 |
| R3 | ヒアリング | 佐々木TL ヒアリング（2026-04-01） | 2026-04-01 | 制約の優先度を確認 |
| R4 | 文書 | 就業規則 第○条 | YYYY-MM-DD | 夜勤後の休息時間 |
| R5 | メール | 田中課長 メール（2026-04-03） | 2026-04-03 | 夜勤人数の緩和可否 |
| R6 | 仮定 | 分析者の仮定 | YYYY-MM-DD | 実データなし |

## 目的

- 主目的: [例: 配送コスト最小化]  ← Ref: R3
- 副目的: [例: ドライバー間の公平性]  ← Ref: R3
- 優先順位: [主目的 > 副目的]  ← Ref: R3

## ハード制約（必ず守る）

| # | 制約 | 根拠 | Ref | 備考 |
|---|------|------|-----|------|
| H1 | [例: 車両積載量 1000kg 以下] | 物理的制約 | R2 | |
| H2 | [例: 営業時間 8:00-18:00] | 就業規則 | R4 | |
| H3 | [例: 夜勤翌日朝勤禁止] | 労働安全 | R4 | 就業規則 第○条 |

## ソフト制約（できれば守りたい）

| # | 制約 | 重み | Ref | 備考 |
|---|------|------|-----|------|
| S1 | [例: 連続稼働 8h 以内] | 高 | R3 | ヒアリングで「絶対ではない」と確認 |
| S2 | [例: 同一エリアは同一ドライバー] | 中 | R3 | |

## 仮定（データがないため仮の値を置いたもの）

| # | 仮定 | 値 | 根拠 | Ref | 検証状況 |
|---|------|-----|------|-----|---------|
| A1 | [例: 平均移動速度] | 30km/h | 一般的な市街地速度 | R6 | 未検証 |
| A2 | [例: 1件あたり作業時間] | 10分 | ヒアリング | R3 | 確認済 |

## データ

| ファイル | 内容 | Ref | 備考 |
|---------|------|-----|------|
| [例: orders.csv] | 注文データ（N件） | R1 | |
| [例: vehicles.csv] | 車両マスタ（M台） | R2 | |

## 受け入れ要件

### 現場の管理方法
- 現在のツール: [例: Excel / 紙 / ホワイトボード / 専用ソフト]  ← Ref: R3
- 操作する人: [例: 管理者のみ / 現場の担当者も]
- 今のツールの良い点: [例: 自由に書き換えられる]
- 今のツールの不満: [例: 作成に3時間かかる、ミスが多い]

### 結果の表示
- 希望する形式: [例: ガントチャート / 地図 / 一覧表 / 既存Excelの形式]
- 閲覧環境: [例: PC / スマホ / 印刷]

### 手修正
- ソルバー出力の手修正: [あり / なし]
- 修正の頻度・理由: [例: 急な欠勤で週1-2回差し替え]
- 修正後の制約チェック: [必要 / 不要]

### 承認・配布フロー
- 承認者: [例: 課長が確認 → 全員に配布]
- 配布方法: [例: メール / 掲示 / システム登録]

### 更新頻度
- 作成頻度: [例: 週1 / 毎日 / 月1]
- 急な変更への対応: [例: 当日の欠勤、急ぎの注文]

### 納品形態
- [例: Excel書き戻し / PDF / Webアプリ / 既存システム連携（API）]

## 前バージョンからの変更点

（v1 では省略。v2 以降で記載する）

| 項目 | 変更前 | 変更後 | 理由 | Ref |
|------|-------|-------|------|-----|
| [例: A1 移動速度] | 30km/h | 22km/h | GPSログ実測値 | R7 |
```
